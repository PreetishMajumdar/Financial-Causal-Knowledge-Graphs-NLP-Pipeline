## 📑 Project Overview

This project delivers an automated Natural Language Processing (NLP) pipeline designed to extract explicit cause-and-effect relationships from unstructured financial policy reports (e.g., Bank of England Financial Stability Reports). The final output is a **Financial Causal Knowledge Graph (FCKG)** that visualizes systemic risks, economic drivers, and policy impacts to support advanced financial modeling.

## 🏗️ Pipeline Architecture

The system operates in four distinct, decoupled stages:

### Stage 1: PDF Parsing & Preprocessing
- **Tool:** `pdfplumber`, `spaCy`
- **Function:** Parses double-column financial PDFs, removes headers/footers/tables, and segments the raw text into clean, individual sentences.
- **Output:** `ALL_reports_sentences.csv`

### Stage 2: The Causal Gatekeeper
- **Tool:** FinBERT (Hugging Face), PyTorch
- **Function:** Solves the "label noise" problem in standard financial datasets. Uses a hybrid approach combining **Domain-Specific Regex Heuristics** with a fine-tuned FinBERT classifier (trained with Focal Loss to handle class imbalance) to filter out noise and only pass strictly causal sentences downstream.
- **Output:** `causal_sentences.csv`

### Stage 3: LLM Relation Extraction
- **Tool:** Google Gemini 3.1 Flash Lite API, `pydantic`
- **Function:** Processes the filtered causal sentences in batches of 10 to optimize Free-Tier API quotas. Uses strict zero-shot prompting to extract `{Source, Target, Polarity}` while preventing LLM hallucination.
- **Output:** `financial_graph_edges.json`

### Stage 4: FCKG Construction & Analytics
- **Tool:** `NetworkX`, `PyVis`
- **Function:** Resolves entity synonyms (e.g., `"Housing Costs"` → `"House Prices"`), aggregates edge frequencies for connection weights, calculates PageRank Centrality to find core systemic drivers, and renders an interactive web graph.
- **Output:** `Pangura_FCKG_Interactive.html` & `Pangura_FCKG_Final.gexf`

## ⚙️ Installation & Setup

**1. Clone the repository and navigate to the folder:**
```bash
git clone https://github.com/PreetishMajumdar/Financial-Causal-Knowledge-Graphs-NLP-Pipeline.git
```

**2. Install the required Python libraries:**

Ensure you are using **Python 3.10+**.
```bash
pip install pandas numpy torch transformers scikit-learn google-genai python-dotenv networkx pyvis pdfplumber spacy
```

**3. Download the spaCy English model (Required for Stage 1):**
```bash
python -m spacy download en_core_web_sm
```

**4. Setup your Environment Variables:**

Create a file named exactly `.env` in the root folder and add your Gemini API key:
```
GEMINI_API_KEY=your_api_key_here
```

## ⚠️ Known Limitations & Future Work

- **Data Scarcity:** The public FinCausal dataset contains significant label noise. Our hybrid Gatekeeper mitigates this, but a purely human-annotated "Gold Standard" dataset would improve precision.
- **Semantic Resolution:** Currently, Entity Resolution (Stage 4) relies on manual mapping dictionaries. Future iterations should implement Cosine Similarity via Vector Embeddings for automated synonym matching.
