## 📑 Project Overview

This project delivers an automated Natural Language Processing (NLP) pipeline designed to extract explicit cause-and-effect relationships from unstructured financial policy reports (e.g., Bank of England Financial Stability Reports). The final output is a **Financial Causal Knowledge Graph (FCKG)** that visualizes systemic risks, economic drivers, and policy impacts to support advanced financial modeling.

## 🏗️ Pipeline Architecture

The system operates in four distinct, decoupled stages:

### Stage 1: PDF Parsing & Preprocessing
- **Tool:** `pdfplumber`, `spaCy`
- **Function:** Parses double-column financial PDFs, removes headers/footers/tables, and segments the raw text into clean, individual sentences.
- **Output:** `ALL_reports_sentences.csv`

### Stage 2: The Causal Gatekeeper
- **Tool:** FinBERT (Hugging Face), PyTorch
- **Function:** Solves the "label noise" problem in standard financial datasets. Uses a hybrid approach combining **Domain-Specific Regex Heuristics** with a fine-tuned FinBERT classifier (trained with Focal Loss to handle class imbalance) to filter out noise and only pass strictly causal sentences downstream.
- **Output:** `causal_sentences.csv`

### Stage 3: LLM Relation Extraction
- **Tool:** Google Gemini 3.1 Flash Lite API, `pydantic`
- **Function:** Processes the filtered causal sentences in batches of 10 to optimise Free-Tier API quotas. Uses an **Ontology-Aware zero-shot prompt** that instructs Gemini to generalise specific textual events into concise, standard financial concepts (1–4 words, e.g., `"a sudden massive spike in crude oil costs"` → `"Oil Prices"`), preventing verbose verbatim phrases from polluting the graph. Extracts `{Cause, Effect, Polarity}` triples with strict schema validation via `pydantic`.
- **Output:** `financial_graph_edges.json`

### Stage 4: FCKG Construction & Analytics
- **Tool:** `NetworkX`, `PyVis`, `sentence-transformers`, `scikit-learn`
- **Function:** Performs **AI-powered Entity Resolution** using `all-MiniLM-L6-v2` vector embeddings and cosine similarity clustering (threshold: 0.85) to automatically merge semantically equivalent entities (e.g., `"Interest Rate"` and `"Interest Rates"`). Removes self-loops, aggregates edge frequencies for connection weights, calculates PageRank Centrality to surface core systemic drivers, and renders an interactive web graph.
- **Output:** `Pangura_FCKG_Interactive.html` & `Pangura_FCKG_Final.gexf`

## ⚙️ Installation & Setup

**1. Clone the repository:**

```bash
git clone https://github.com/PreetishMajumdar/Financial-Causal-Knowledge-Graphs-NLP-Pipeline.git
```

**2. Install the required Python libraries:**

Ensure you are using **Python 3.10+**.

```bash
pip install pandas numpy torch transformers scikit-learn sentence-transformers google-genai python-dotenv networkx pyvis pdfplumber spacy
```

**3. Download the spaCy English model (Required for Stage 1):**

```bash
python -m spacy download en_core_web_sm
```

**4. Setup your Environment Variables:**

Create a file named exactly `.env` in the root folder and add your Gemini API key:
GEMINI_API_KEY=your_api_key_here

## 🚀 How to Run the Pipeline

### Step 1: Extract Text from PDFs
Place your financial reports in the `Reports/` folder.
```bash
python stage1_pdf_parser.py
```

### Step 2: Train & Filter Causal Sentences
> **Note:** If you have already trained the model, comment out the `gatekeeper.train_model()` line in the main block of the script to save time and skip directly to inference.
```bash
python stage2_causal_gatekeeper.py
```

### Step 3: Extract Graph Edges (Gemini API)
> This script runs in batches of 10 with a 5-second delay between each batch to respect API rate limits. Gemini is prompted to output concise 1–4 word financial concepts rather than verbatim sentence fragments.
```bash
python stage3_extractor.py
```

### Step 4: Build & Visualize the Graph
> This step downloads the `all-MiniLM-L6-v2` embedding model on first run (~80 MB). Subsequent runs use the cached version.
```bash
python stage4_graph_builder.py
```

Open `Financial Causal Knowledge Graph/Pangura_FCKG_Interactive.html` in your web browser to view the interactive visualization!

## ⚠️ Known Limitations & Future Work

- **Data Scarcity:** The public FinCausal dataset contains significant label noise. Our hybrid Gatekeeper mitigates this, but a purely human-annotated "Gold Standard" dataset would further improve precision.
- **Embedding Clustering Sensitivity:** The cosine similarity threshold for entity resolution (currently `0.85`) may require tuning depending on the vocabulary of the source reports. A threshold that is too low risks merging distinct concepts; too high leaves duplicates in the graph.
- **API Dependency:** Stage 3 relies on the Gemini Free Tier, which enforces rate limits. For large report corpora, consider upgrading to a paid tier or implementing a local LLM fallback.
